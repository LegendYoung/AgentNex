# AgentNex Platform P0 前端功能测试指南

## 测试环境准备

### 1. 启动后端服务
```bash
cd d:\AgentNex_Platform\AgentNex
docker-compose up -d
```

### 2. 启动前端开发服务器
```bash
cd apps/web
pnpm dev
```

### 3. 访问应用
- 前端地址：http://localhost:5173
- API 地址：http://localhost:8000
- API 文档：http://localhost:8000/docs

### ✅ 测试 2：Agent 创建向导

#### 2.1 创建 Agent - 步骤1：基础配置
1. 点击侧边栏 "Agent" 菜单
2. 点击右上角 "新建 Agent" 按钮
3. 填写 Agent 名称（2-32字符）
4. 填写描述（可选）
5. 填写系统提示词（10-5000字符，支持 Markdown）
6. 选择 LLM 模型（默认 qwen-plus）
7. 调整 Temperature 和 Top-P 参数
8. 点击 "预览效果" 查看提示词拼接
9. 点击 "暂存草稿" 测试草稿保存
10. 点击 "下一步"
11. ✅ 预期：所有输入项校验生效，可进入下一步

#### 2.2 创建 Agent - 步骤2：能力开关
1. **记忆功能**
   - 开启记忆功能开关
   - 选择记忆类型（短期/长期）
   - 调整记忆窗口（1-50轮）
2. **知识库**
   - 开启知识库开关
   - 勾选已有知识库（支持多选）
3. **工具调用**
   - 开启工具调用开关
   - 勾选内置工具（网页搜索、文件读取、Python执行）
   - 设置权限控制
4. 点击 "下一步"
5. ✅ 预期：能力开关配置生效，可进入测试步骤

#### 2.3 创建 Agent - 步骤3：实时测试
1. 在输入框输入测试问题
2. 点击 "发送" 或按 Enter
3. 查看 Agent 回复
4. 展开查看工具调用日志
5. 展开查看知识库检索结果
6. 展开查看记忆调用内容
7. 点击 "清空对话" 重新测试
8. 点击 "下一步"
9. ✅ 预期：可测试 Agent 效果，查看完整运行日志

#### 2.4 创建 Agent - 步骤4：发布确认
1. 查看配置摘要
2. 选择可见性：
   - "仅自己可见"
   - "指定团队共享"（选择团队和权限）
3. 点击 "发布 Agent"
4. ✅ 预期：Agent 创建成功，跳转到详情页

### ✅ 测试 4：知识库管理
